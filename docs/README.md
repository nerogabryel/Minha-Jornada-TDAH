# Documentação do Projeto - Minha Jornada TDAH

Este diretório contém o histórico de mudanças, decisões arquiteturais e evoluções do projeto. O objetivo é manter um registro claro do "O que", "Como" e "Por que" as mudanças foram feitas, facilitando a manutenção futura e o onboarding de novos desenvolvedores.

## Estrutura

- **historico/**: Contém registros cronológicos das grandes alterações e sprints de desenvolvimento.

## Convenção de Logs

Cada arquivo de log deve seguir o formato `XXX_titulo_da_mudanca.md` e conter:
1. **Contexto**: O estado anterior ou o problema a ser resolvido.
2. **Mudanças**: O que foi feito tecnicamente.
3. **Justificativa**: Por que essa abordagem foi escolhida.
4. **Lições Aprendidas**: O que evitar ou melhorar no futuro.
